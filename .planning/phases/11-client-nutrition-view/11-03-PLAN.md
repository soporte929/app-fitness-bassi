# Plan 11.3: Free Food Logging & Diary

## Objective
Implement a modal or separate page where the client can search for a food in the database, specify the grams eaten, and log it to their daily diary (`food_log` table). This implements CNUTR-05.

## Context
- `.planning/SPEC.md` (or ROADMAP.md Phase 11)
- `lib/supabase/types.ts`
- `app/(client)/nutrition/page.tsx`

## Tasks

<task type="auto">
  <name>Food Search & Selection Modal</name>
  <files>
    - components/client/nutrition/FoodSearchModal.tsx
    - app/(client)/nutrition/actions.ts
    - app/(client)/nutrition/page.tsx
  </files>
  <action>
    - Implement a `FoodSearchModal` (or sliding drawer) accessible from the Nutrition Dashboard.
    - Create a server action `searchFoodsAction(query: string)` that searches the `foods` table (case insensitive, partial match on name).
    - Display results in a list inside the modal with their macros per 100g.
  </action>
  <verify>Check if searching e.g., 'Pollo' returns 'Pechuga de pollo' from seeds</verify>
  <done>Client can successfully surface any seed food using the search input</done>
</task>

<task type="auto">
  <name>Dynamic Grams Input & Logging</name>
  <files>
    - components/client/nutrition/FoodSearchModal.tsx
    - app/(client)/nutrition/actions.ts
  </files>
  <action>
    - Once a food is selected from the search list, present a number input for `grams`.
    - As the client types the grams, instantly calculate and display the extrapolated macros (pro, fat, car, kcal).
    - Create a server action `logFreeFoodAction(clientId, foodId, grams, date)` that securely inserts a row into `food_log`.
    - Upon successful insert, close the modal, show a toast, and call `revalidatePath('/(client)/nutrition')` to update the macro bars.
  </action>
  <verify>Manually insert 200g of a food and ensure the progress bars update correctly</verify>
  <done>Client can enter custom grams and immediately see the food added to their daily totals</done>
</task>

## Success Criteria
- [ ] User can open the meal logger arbitrarily.
- [ ] Searching the food database is responsive and functional.
- [ ] Custom macro calculation is accurate inside the modal.
- [ ] Successful logging impacts the macro progress bars on the root page instantly.
