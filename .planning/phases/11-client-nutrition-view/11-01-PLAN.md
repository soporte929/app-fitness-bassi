# Plan 11.1: Client Dashboard Layout & Macro Progress Bars

## Objective
Implement the main `/nutrition` page for clients. Fetch the active nutrition plan, fetch today's logged food entries, and build the top-level progress bars (calories and macros) showing consumption vs daily targets.

## Context
- `.planning/SPEC.md` (or ROADMAP.md Phase 11)
- `lib/supabase/types.ts` (for `nutrition_plans`, `food_log`, `foods`)
- `app/(client)/nutrition/page.tsx`

## Tasks

<task type="auto">
  <name>Server Action for Client Nutrition Context</name>
  <files>
    - app/(client)/nutrition/actions.ts
  </files>
  <action>
    - Create an action `getClientNutritionContextAction(clientId: string, dateStr: string)`
    - It must fetch the active `nutrition_plans` for the client.
    - It must fetch `food_log` for the given date (joined with `foods` to get macros).
    - It must return the plan targets and the summed consumed macros from `food_log`.
    - Avoid `as any` types. Return a strictly typed result.
  </action>
  <verify>Check TS types for the server action return type</verify>
  <done>Server action successfully queries plan and daily food log with sums</done>
</task>

<task type="auto">
  <name>Nutrition Page Layout & Progress Bars</name>
  <files>
    - app/(client)/nutrition/page.tsx
    - components/client/nutrition/MacroProgressBars.tsx
  </files>
  <action>
    - Implement the client nutrition page `page.tsx` that calls `getClientNutritionContextAction` for the current logged-in user and today's date.
    - Create `MacroProgressBars.tsx` showing 4 progress bars: Calories, Protein, Fat, Carbohydrates.
    - Display the consumed vs target values (e.g., "1500 / 2000 kcal").
    - Calculate percentage for the bars. Use the target from `nutrition_plans`. If no plan is active, show an empty state or message.
  </action>
  <verify>Run the app and navigate to client nutrition page</verify>
  <done>The client sees 4 progress bars comparing their consumed amounts against daily targets</done>
</task>

## Success Criteria
- [ ] Client can access `/nutrition` and retrieve their assigned active plan.
- [ ] Progress bars display correctly and calculate percentages without crashing.
- [ ] TypeScript compilation succeeds with strict typings.
