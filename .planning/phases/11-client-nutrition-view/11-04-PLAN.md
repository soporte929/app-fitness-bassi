# Plan 11.4: Weekly Shopping List Generation

## Objective
Implement a view or modal where the client can see an automatically generated weekly shopping list based on their assigned active nutrition plan. This implements CNUTR-06.

## Context
- `.planning/SPEC.md` (or ROADMAP.md Phase 11)
- `lib/supabase/types.ts`
- `app/(client)/nutrition/page.tsx`
- `app/(client)/nutrition/shopping-list/page.tsx`

## Tasks

<task type="auto">
  <name>Shopping List Calculation Logic</name>
  <files>
    - app/(client)/nutrition/actions.ts
    - app/(client)/nutrition/shopping-list/page.tsx
  </files>
  <action>
    - Create a page route `/shopping-list` inside the nutrition feature domain, or a modal accessible from the dashboard.
    - Write a server action `generateWeeklyShoppingListAction(clientId: string)` that fetches the client's current `multi_plan_items` from their active plan.
    - Loop over all the `meal_plan_items` for Option 'A' (by default) or sum them across all meals. Group by `food_id` and sum their `grams`.
    - Multiply each grouped sum by 7 to determine the required grams for a full 7-day week.
    - Return a structurally typed array `{ food_name: string, category: string, total_grams: number }`.
  </action>
  <verify>Call action and confirm the items are correctly aggregated and multiplied by 7</verify>
  <done>Backend action calculates the correct total amounts for a structured diet week</done>
</task>

<task type="auto">
  <name>Shopping List UI & Category Grouping</name>
  <files>
    - app/(client)/nutrition/shopping-list/page.tsx
  </files>
  <action>
    - Create a polished list UI taking the array from the action.
    - Group the returned items visually by their source table `category` (e.g., "Proteínas", "Carbohidratos").
    - Format quantities appropriately (e.g., if > 1000g, format as `1.2 kg`, otherwise `250 g`).
    - Include a clear button/header for navigation back to the Dashboard.
  </action>
  <verify>Navigate to the shopping list sub-route and verify categories and UI design</verify>
  <done>Client can see an organized layout with their required grocery amounts</done>
</task>

## Success Criteria
- [ ] Summation logic properly handles multiple occurrences of the same food (e.g., chicken at lunch and dinner).
- [ ] Gram conversion (using kg when appropriate) works flawlessly.
- [ ] No `any` casting in TS definitions.
