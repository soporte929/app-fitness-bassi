# Plan 39.2: Fix modal "Añadir ejercicio" — posición, filtros y cierre

<objective>
Corregir tres problemas del modal ExercisePicker en producción:
1. **Posición incorrecta**: el modal aparece demasiado arriba, no centrado correctamente en la viewport.
2. **Filtros cortados**: cuando el filtro activo es "Todos", la barra horizontal de chips de grupos musculares se corta visualmente (overflow oculto o altura insuficiente).
3. **No se puede cerrar**: clicking outside del modal no cierra correctamente — el layout actual separa el backdrop del contenedor flex, dejando zonas muertas donde los clicks no llegan al handler `onClose`.

Purpose: Restaurar la usabilidad del principal punto de entrada para agregar ejercicios a una rutina.
Output: `exercise-picker.tsx` con layout corregido.
</objective>

  <name>Fix modal layout, filter overflow, and close behavior</name>
  <files>components/trainer/exercise-picker.tsx</files>
  <action>
    Diagnosticar y corregir los tres sub-bugs en el componente ExercisePicker:

    **1. Posición del modal**
    El modal actual usa `fixed inset-0 flex items-end md:items-center justify-center` como contenedor externo (línea 136-140). El problema es que este div tiene `pointer-events-none` y solo pasa a `pointer-events-auto` cuando `open=true`, pero no actúa como backdrop — el backdrop es un div hermano anterior que cubre `fixed inset-0`.

    FIX: Fusionar el backdrop y el contenedor de posicionamiento en UN solo div con `fixed inset-0`. Este div único:
    - Actúa como backdrop (bg-black/40) con `onClick={onClose}`
    - Contiene el centering flex (items-end md:items-center justify-center)
    - Solo se renderiza / tiene pointer-events cuando `open=true`

    El div interno del modal (línea 142-152) mantiene `e.stopPropagation()` para evitar que clicks dentro del modal cierren el backdrop.

    **2. Filtros cortados**
    La barra de filter chips (líneas 185-213) usa `overflow-x-auto` pero puede quedarse sin espacio vertical dentro del `max-h-[82vh] flex flex-col`. Con muchos chips (10 grupos musculares), el contenedor no reserva suficiente espacio.

    FIX: 
    - Añadir `flex-shrink-0` a la sección de filter chips para que no se comprima.
    - Asegurar que el `flex-shrink-0` también está en el header y el search bar (ya podrían tenerlo implícitamente por ser contenido fijo).
    - Solo el listado de ejercicios (`flex-1 overflow-y-auto`) debe comprimirse.

    **3. Close behavior**
    Con el fix del punto 1 (backdrop unificado), el cierre al clickar fuera queda resuelto automáticamente. Verificar además:
    - El botón X (línea 161-167) tiene `onClick={onClose}` — OK.
    - `handleSelect` llama `onClose()` — OK.
    - Asegurar que NO hay zonas donde un click caiga en "tierra de nadie" sin handler.

    AVOID: No cambiar la API del componente (Props). No tocar LIBRARY ni la lógica de filtrado, solo el layout/CSS.
  </action>
