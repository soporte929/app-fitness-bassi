# Plan 39.1: Fix state drops in RoutineBuilder wizard

## Objective
Avoid the loss of focus and component internal state (like `expanded` in `ExerciseRow` and `editing` in `EditableDayName`) when switching between steps in the `RoutineBuilder` multi-step wizard.

## Context
- .planning/ROADMAP.md
- components/trainer/routine-builder.tsx

## Tasks

<task type="auto">
  <name>Change conditional rendering to CSS display</name>
  <files>components/trainer/routine-builder.tsx</files>
  <action>
    Modify the `RoutineBuilder` so it does not unmount step content. Currently, steps are rendered conditionally (`{currentStep === 1 && (<div.../>)}`). This destroys internal component state (like `expanded` or active cursors) whenever the step changes.
    Instead, render all steps simultaneously but hide them with CSS when inactive.
    
    1. For Step 1: Wrap its contents in `<div className={currentStep === 1 ? 'block' : 'hidden'}>` and remove the `currentStep === 1 &&` condition.
    2. For Step 2: Use `<div className={currentStep === 2 && !structureLocked ? 'block' : 'hidden'}>`
    3. For Step 3: Use `<div className={currentStep === 3 && !structureLocked ? 'block' : 'hidden'}>`
    
    Ensure `cn` handles the `hidden` class correctly if merged with `space-y-4` or similar existing container classes.
  </action>
  <verify>npm run check (to ensure TypeScript is valid without TS errors)</verify>
  <done>Navigating through steps in the form preserves local component state like `expanded` in `ExerciseRow` because elements are not unmounted.</done>
</task>

## Success Criteria
- [ ] No state drops navigating back and forth between Info básica, Ejercicios, and Días.
- [ ] Exercise expanded states survive moving back to step 1 and then to step 2 again.
