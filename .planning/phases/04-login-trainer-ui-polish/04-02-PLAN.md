  <action>
El `ThemeProvider` actual inicializa el estado React en `useState` pero no ejecuta `applyTheme` en el mount. Esto significa que si localStorage tiene `theme=light`, el script inline de layout.tsx ya habrá seteado `data-theme="light"` en el DOM, pero el estado React podría no estar sincronizado tras la hidratación (porque `document.documentElement.classList.contains('dark')` puede retornar un valor inesperado dependiendo del timing).

Cambio requerido: añadir un `useEffect` con dependencia `[]` que:
1. Lea `localStorage.getItem('theme')` para obtener el tema guardado
2. Determine el tema final: si hay valor guardado, usarlo; si no, 'dark'
3. Llame a `applyTheme(finalTheme)` para garantizar que DOM y estado React coinciden
4. Llame a `setThemeState(finalTheme)` para actualizar el estado React

Además, añadir `useEffect` al import de React (actualmente el archivo importa `createContext, useContext, useState` de 'react').

El archivo resultante debe:
- Importar `createContext, useContext, useState, useEffect` de 'react'
- Mantener todas las funciones existentes (`applyTheme`, `setTheme`, `toggleTheme`) sin cambios
- Añadir un `useEffect(() => { ... }, [])` dentro de `ThemeProvider` que sincroniza el tema al montar

Estructura del useEffect a añadir (dentro de `ThemeProvider`, después de los `useState` y antes del `return`):
```typescript
useEffect(() => {
  let saved: Theme = 'dark'
  try { saved = (localStorage.getItem('theme') as Theme | null) ?? 'dark' } catch { /* ignore */ }
  applyTheme(saved)
  setThemeState(saved)
}, [])
```

NO cambiar la firma del contexto, NO cambiar setTheme, NO cambiar toggleTheme.
