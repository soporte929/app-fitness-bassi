# Phase 44 - Plan 3 (Gap Closure)
## Fix ExercisePicker desktop height

**Context**: 
El modal `ExercisePicker` había sido corregido para dispositivos móviles estableciendo la altura a `75dvh`. Sin embargo, en el breakpoint de desktop (`min-[431px]`), seguía aplicando `max-h-[85vh]`. Esto causaba que el modal apareciera cortado por arriba, ocultando tanto el header ("Añadir ejercicio") como el botón de cierre (X), además de parte del buscador.

**Objetivo**:
Asegurar que el header, la barra de búsqueda y los filtros del `ExercisePicker` siempre estén visibles tanto en desktop como en mobile, cortando únicamente el listado de ejercicios.

**Tasks**:
1. Modificar `components/trainer/exercise-picker.tsx`.
2. Remover la regla `min-[431px]:max-h-[85vh]` y aplicar `max-h-[75dvh]` globalmente para simplificar y homogeneizar el comportamiento que previene que se corte el contenedor.
3. Verificar que los elementos superiores (header, buscador de ejercicios, row de filtros por grupo muscular) incluyen `flex-shrink-0` para garantizar que permanecen intactos.

**Success Criteria**:
- El header con el texto "Añadir ejercicio" y la "X" son claramente visibles en pantallas grandes sin hacer scroll.
- El dropdown/modal nunca excede un alto razonable de viewport.
