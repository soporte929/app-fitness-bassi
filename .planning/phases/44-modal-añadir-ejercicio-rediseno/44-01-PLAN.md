# Plan 44.1: Rediseño modal ExercisePicker — bottom sheet mobile / modal desktop

<objective>
Rediseñar el componente `ExercisePicker` para que se comporte como bottom sheet nativo en viewports móviles (≤430px) y como modal centrado en viewports de escritorio (>430px).

Purpose: El modal actual usa `md:` (768px) como breakpoint, pero la app se usa principalmente en móviles con max-width de ~430px. Al cambiar a 430px como punto de corte, el comportamiento en dispositivos reales será correcto. Además, se debe asegurar que no haya overflow, clipping, ni problemas de cierre.

Output: `exercise-picker.tsx` rediseñado con breakpoint correcto, layout robusto y cierre fiable.
</objective>

<task type="auto">
  <name>Cambiar breakpoint de md (768px) a 430px y reforzar layout</name>
  <files>components/trainer/exercise-picker.tsx</files>
  <action>
    Refactorizar el layout del ExercisePicker con estos cambios concretos:

    **1. Breakpoint 430px en lugar de md (768px)**
    Reemplazar todas las clases `md:` del componente por `min-[431px]:` (Tailwind arbitrary breakpoint).
    Mapeo exacto:
    - `md:items-center` → `min-[431px]:items-center`
    - `md:p-4` → `min-[431px]:p-4`
    - `md:max-w-md` → `min-[431px]:max-w-md`
    - `md:rounded-xl` → `min-[431px]:rounded-xl`
    - `md:hidden` (drag handle) → `min-[431px]:hidden`

    **2. Bottom sheet en mobile (≤430px)**
    En viewports ≤430px:
    - El modal debe anclarse al fondo de la pantalla (`items-end` por defecto, ya correcto)
    - Bordes redondeados solo arriba: `rounded-t-xl` (ya correcto)
    - Max height: `max-h-[85vh]` para dejar margen superior visible (en lugar del actual 82vh)
    - Drag handle visible: la barra gris de 40px × 4px ya existe, solo verificar que se muestra
    - Padding inferior: añadir `pb-[env(safe-area-inset-bottom,0px)]` al container del modal para evitar que el home indicator de iOS tape contenido

    **3. Modal centrado en desktop (>430px)**
    En viewports >430px:
    - El modal se centra vertical y horizontalmente (`min-[431px]:items-center justify-center`)
    - Max width: `min-[431px]:max-w-md` (~448px)
    - Max height: `max-h-[85vh]` (mismo)
    - Bordes redondeados completos: `min-[431px]:rounded-xl`

    **4. Prevenir overflow y clipping**
    - Asegurar que TODAS las secciones fijas (header, search, filter chips) tienen `flex-shrink-0`
    - Solo la lista de ejercicios (`div` con el map de filtered) tiene `flex-1 overflow-y-auto`
    - Los filter chips usan `overflow-x-auto` + `scrollbar-none` (ya presente)
    - Añadir `min-h-0` al contenedor flex del modal para que el overflow funcione correctamente en flex layouts (Firefox fix)

    **5. Animación de entrada/salida**
    - Mobile: slide up desde abajo (`translate-y-full` → `translate-y-0`)
    - Desktop: fade + scale (`scale-95 opacity-0` → `scale-100 opacity-100`)
    - Usar clases distintas por breakpoint:
      En móvil (default): `translate-y-full` cuando cerrado, `translate-y-0` cuando abierto
      En desktop: `min-[431px]:translate-y-0 min-[431px]:scale-95` cuando cerrado, `min-[431px]:scale-100` cuando abierto

    **6. Cierre fiable**
    - Backdrop: `onClick={onClose}` en el div externo unificado (ya implementado desde 39-02)
    - X button: `onClick={onClose}` (ya correcto)
    - Selección de ejercicio: `onClose()` en `handleSelect` (ya correcto)
    - Contenido: `e.stopPropagation()` en el div interno (ya correcto)
    - Añadir `onKeyDown` handler para cerrar con Escape:
      ```tsx
      useEffect(() => {
        if (!open) return
        function handleEsc(e: KeyboardEvent) {
          if (e.key === 'Escape') onClose()
        }
        document.addEventListener('keydown', handleEsc)
        return () => document.removeEventListener('keydown', handleEsc)
      }, [open, onClose])
      ```
    
    AVOID: 
    - No cambiar la API del componente (Props siguen siendo {open, onClose, onSelect})
    - No tocar LIBRARY ni la lógica de filtrado (useMemo, query, filter)
    - No cambiar handleSelect
    - No usar @media queries en CSS externo; resolver todo con Tailwind inline
    - No introducir dependencias nuevas (framer-motion, etc.)
  </action>
