# Plan 24.1: Fix prefix collision in middleware

## Objective
Fix the middleware routing bug where prefix overlaps (`/routines` and `/routines-templates`, `/nutrition` and `/nutrition-plans`) incorrectly redirect trainers to the dashboard (BUG-01, BUG-02).

## Context
- `ROADMAP.md`
- `middleware.ts`

## Tasks

<task type="auto">
  <name>Fix middleware routing logic</name>
  <files>middleware.ts</files>
  <action>
    Review and modify middleware.ts to prevent false positive matches.
    Specifically, ensure that `isClientRoute` checks for exact matches or uses a precise prefix mapping (e.g., `pathname === "/routines" || pathname.startsWith("/routines/")`) rather than just `pathname.startsWith("/routines")` which catches `/routines-templates`.
    Ensure the `isTrainerRoute` check securely routes trainers before the client catch-all logic, preventing them from being swept up by `/nutrition-plans`.
  </action>
  <verify>
    grep -E "pathname === \"/routines\"|pathname === \"/nutrition\"" middleware.ts
  </verify>
  <done>
    `middleware.ts` correctly differentiates client `/routines` and `/nutrition` from trainer `/routines-templates` and `/nutrition-plans`, ensuring trainers don't get redirected.
  </done>
</task>

## Success Criteria
- [ ] Trainer can navigate to `/routines-templates` without redirection.
- [ ] Trainer can navigate to `/nutrition-plans` without redirection.
