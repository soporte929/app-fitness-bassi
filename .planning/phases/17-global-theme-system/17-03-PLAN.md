# Plan 17.3: Replace hardcoded dark colors — Nutrition & Trainer modules

<objective>
Eliminate all remaining hardcoded dark hex colors from the nutrition client pages, nutrition components, and trainer plans/nutrition-plans pages. This is the final sweep needed for the theme toggle to work end-to-end.

Purpose: Plans 17.1 + 17.2 handle the infrastructure and layout files. This plan handles the remaining ~30 instances across the nutrition module and trainer admin pages.
Output: Zero hardcoded dark colors in the entire codebase. Light mode works fully.
</objective>

<context>
Load for context:
- app/globals.css (CSS variable definitions for both themes)
- The 12 files listed in files_modified
</context>

<tasks>

<task type="auto">
  <name>Fix client nutrition pages and components</name>
  <files>
    app/(client)/nutrition/page.tsx
    app/(client)/nutrition/nutrition-checklist.tsx
    app/(client)/nutrition/shopping-list/page.tsx
    components/client/nutrition/ClientDailyMeals.tsx
    components/client/nutrition/MacroProgressBars.tsx
  </files>
  <action>
    **Apply consistent mapping across all files:**
    - `bg-[#212121]` → `bg-[var(--bg-surface)]`
    - `border-[rgba(255,255,255,0.07)]` → `border-[var(--border)]`
    - `border-[rgba(255,255,255,0.04)]` → `border-[var(--border)]`
    - `border-[rgba(255,255,255,0.05)]` → `border-[var(--border)]`
    - `text-[#e8e8e6]` → `text-[var(--text-primary)]`
    - `text-[#a0a0a0]` → `text-[var(--text-muted)]`
    - `divide-[rgba(255,255,255,0.04)]` → `divide-[var(--border)]`

    These are all Tailwind arbitrary value classes — the replacement is a direct string substitution within each className.

    AVOID: Do NOT change any layout structure, padding, margin, or border-radius values.
    AVOID: Do NOT touch accent colors like `#fb8500` (orange CTA) — those are intentional brand colors not affected by theme.
  </action>
  <verify>Toggle theme — nutrition pages should show correct surfaces/borders in both modes.</verify>
  <done>All 5 client nutrition files use CSS variables. No hardcoded hex backgrounds remain.</done>
</task>
