# Plan 38.1: Remove client-mode code from Routine Templates module

<objective>
Purge all remaining `mode === 'client'` logic from the routine templates subsystem.

Purpose: Phase 30 removed the direct-assignment buttons/modals (AssignRoutineButton, AssignTemplateModal, clonePlanToClientAction). However, the RoutineBuilder component and the /routines-templates pages still contain full `mode=client` support — including a "Plan para cliente" button that escaped Phase 31's cleanup. This creates confusion and violates the strict Rutina → Plan → Cliente flow. A trainer should only create templates from /routines-templates; assigning routines to clients happens exclusively through the Plans module (/plans).

Output: 6 modified files with all client-mode remnants stripped. The RoutineBuilder only creates templates. The page only shows templates.
</objective>
