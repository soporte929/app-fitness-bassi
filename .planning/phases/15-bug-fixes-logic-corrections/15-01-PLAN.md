    El input de búsqueda en MealSlot pierde foco porque el re-render del componente padre destruye y recrea el input en cada keystroke.

    Solución: Extraer el input de búsqueda en un sub-componente local dentro de meal-slot.tsx que use `useRef` para preservar el foco.

    En meal-slot.tsx, añadir un componente local ANTES del componente MealSlot:
    ```tsx
    // Sub-componente para preservar foco del input entre re-renders del padre
    function FoodSearchInput({
      value,
      onChange,
      placeholder,
    }: {
      value: string
      onChange: (v: string) => void
      placeholder?: string
    }) {
      const inputRef = useRef<HTMLInputElement>(null)
      return (
        <div className="relative">
          <Search className="absolute left-3 top-1/2 -translate-y-1/2 w-4 h-4 text-[var(--text-muted)]" />
          <input
            ref={inputRef}
            type="search"
            value={value}
            onChange={(e) => onChange(e.target.value)}
            placeholder={placeholder ?? 'Buscar alimento...'}
            className="w-full bg-[var(--bg-base)] border border-[var(--border)] rounded-lg py-2 pl-9 pr-3 text-sm text-[var(--text-primary)] focus:outline-none focus:border-[var(--accent)] placeholder:text-[var(--text-muted)]"
          />
        </div>
      )
    }
    ```

    CRÍTICO: El `key` prop del componente FoodSearchInput NO debe ser dinámico (no usar `key={searchQuery}` o similar). Si el componente existente tiene un `key` que cambia, eliminarlo. Un key dinámico destruye y recrea el componente — anulando el efecto del useRef.

    Luego reemplazar el JSX del input de búsqueda existente dentro de MealSlot con `<FoodSearchInput value={searchQuery} onChange={setSearchQuery} />`.

    Añadir `useRef` a los imports de React si no está ya importado.
