# Plan 18.1: Subida de foto de perfil

## Objective
Permitir que el cliente (o cualquier usuario) pueda subir una foto de perfil desde su dispositivo interactuando con la galería/cámara nativa, guardarla en Supabase Storage (bucket `avatars`) y actualizar su registro con la nueva `avatar_url`.

## Context
- `.planning/ROADMAP.md`
- `app/(client)/profile/edit-profile-form.tsx`
- `app/(client)/profile/actions.ts`
- `lib/supabase/client.ts`

<task type="auto">
  <name>Implementar subida de foto en EditProfileForm</name>
  <files>
    app/(client)/profile/edit-profile-form.tsx
  </files>
  <action>
    Modificar `EditProfileForm` para:
    - Ocultar el input tipo `url` para el avatar y sustituirlo por un input oculto `<input type="file" accept="image/*" />` accesible al tocar el avatar o un botón "Cambiar foto".
    - Importar `createClient` de `@/lib/supabase/client`.
    - Al dispararse el `onChange` del archivo, subir la foto a `avatars/<user_id>/<timestamp>` (u otra ruta válida).
    - Usar estado `uploading` para mostrar un spinner en lugar del avatar mientras se sube.
    - Al acabar la subida satisfactoriamente, se genera la public URL usando `supabase.storage.from('avatars').getPublicUrl(path)` y se setea en el estado `avatarUrl`.
    - No eliminar el guardado previo: la URL subida se pasa igualmente al pulsar "Guardar" y usar `updateProfileAction`.
  </action>
  <verify>Chequear que `input type="file"` se renderiza como trigger en la UI y utiliza el cliente Supabase para subir a Storage.</verify>
  <done>La foto de perfil se puede seleccionar desde la galería y subir, viéndose instantáneamente tras el upload, y persiste al presionar Guardar.</done>
</task>

## Success Criteria
- [ ] La migración de base de datos define el bucket de storage "avatars" y permite su uso.
- [ ] El UI de cambiar logo invita a usar la cámara o galería del dispositivo tocando el avatar actual.
- [ ] La subida produce un archivo válido y una `public_url` en base de datos.
