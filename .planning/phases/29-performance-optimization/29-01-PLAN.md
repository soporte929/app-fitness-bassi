En dashboard/page.tsx:

1. Añadir import al top: `import { unstable_cache } from 'next/cache'` y `import { createAdminClient } from '@/lib/supabase/admin'`

2. Crear función `getTrainerDashboardData` fuera del componente, usando `unstable_cache`:

```typescript
const getTrainerDashboardData = unstable_cache(
  async (trainerId: string, thirtyDaysAgo: string) => {
    const supabase = createAdminClient()

    const { data: rawClients } = await supabase
      .from('clients')
      .select(`id, phase, weight_kg, profile:profiles!clients_profile_id_fkey (full_name)`)
      .eq('trainer_id', trainerId)
      .eq('active', true)

    const allClients = rawClients ?? []
    const clientIds = allClients.map((c) => c.id)

    if (clientIds.length === 0) {
      return { clients: allClients, sessions: [], weightLogs: [] }
    }

    const [{ data: sessions }, { data: weightLogs }] = await Promise.all([
      supabase
        .from('workout_sessions')
        .select('client_id, started_at, completed')
        .in('client_id', clientIds)
        .gte('started_at', thirtyDaysAgo),
      supabase
        .from('weight_logs')
        .select('client_id, weight_kg, logged_at')
        .in('client_id', clientIds)
        .gte('logged_at', thirtyDaysAgo)
        .order('logged_at', { ascending: true }),
    ])

    return {
      clients: allClients,
      sessions: sessions ?? [],
      weightLogs: weightLogs ?? [],
    }
  },
  ['trainer-dashboard-data'],
  { revalidate: 60, tags: ['trainer-dashboard'] }
)
```

3. En el componente `TrainerDashboard()`:
   - Mantener `supabase.auth.getUser()` directamente (no cachear auth — siempre fresco)
   - Calcular `thirtyDaysAgo` como antes
   - Reemplazar las queries directas por: `const { clients: allClients, sessions: allSessions, weightLogs: allWeightLogs } = await getTrainerDashboardData(user.id, thirtyDaysAgo.toISOString())`
   - Eliminar las variables `rawClients`, `sessions`, `weightLogs` del componente (ya vienen de la función cacheada)
   - El resto del componente (cálculo de clientStats, charts, render) permanece sin cambios

IMPORTANTE: `createAdminClient()` no recibe cookies — es seguro usarlo dentro de `unstable_cache`. El `trainerId` como argumento garantiza que cada trainer tiene su propia entrada en el cache (Next.js crea keys por combinación de argumentos).

NO cachear `supabase.auth.getUser()` — siempre debe ejecutarse en el request para garantizar autenticación fresca.
