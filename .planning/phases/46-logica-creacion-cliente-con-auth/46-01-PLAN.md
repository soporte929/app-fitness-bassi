Crear el directorio `app/auth/callback/` y el archivo `route.ts`.

El handler GET intercambia el `code` de la URL (token de invitación de Supabase) por una sesión activa, luego redirige al cliente a `/today`.

```typescript
import { createClient } from '@/lib/supabase/server'
import { NextRequest, NextResponse } from 'next/server'

export async function GET(request: NextRequest) {
  const { searchParams, origin } = new URL(request.url)
  const code = searchParams.get('code')

  if (code) {
    const supabase = await createClient()
    await supabase.auth.exchangeCodeForSession(code)
  }

  return NextResponse.redirect(`${origin}/today`)
}
```

Notas de implementación:
- `origin` se extrae de la URL del request para que funcione tanto en local como en producción sin hardcodear URLs
- El redirect destino es `/today` (app cliente) — es donde deben llegar los clientes tras establecer su password
- No se necesita manejo de error explícito: si el exchange falla, el cliente llega a /today sin sesión y el middleware lo redirige a /login
- Importar `createClient` de `@/lib/supabase/server` (SSR con cookies) — no el admin client
