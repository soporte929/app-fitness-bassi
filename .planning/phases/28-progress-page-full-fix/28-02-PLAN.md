  </what-built>
  <how-to-verify>
    Abre Supabase Dashboard → SQL Editor y ejecuta estos pasos en orden:

    **Paso 1 — Verificar policies actuales:**
    ```sql
    SELECT policyname, cmd, qual, with_check
    FROM pg_policies
    WHERE schemaname = 'public' AND tablename = 'client_measurements'
    ORDER BY cmd;
    ```
    Anota qué policies existen para SELECT.

    **Paso 2 — Verificar si hay datos para el cliente de prueba:**
    ```sql
    SELECT * FROM client_measurements
    WHERE client_id = '24646591-53ec-4d1a-b92a-08f00e8d365b'
    ORDER BY measured_at DESC
    LIMIT 10;
    ```

    **Paso 3 — Aplicar fix de RLS (ejecutar siempre, es idempotente):**
    ```sql
    DROP POLICY IF EXISTS "Clients can read own measurements" ON client_measurements;
    DROP POLICY IF EXISTS "clients_read_own" ON client_measurements;
    DROP POLICY IF EXISTS "clients_select_own_measurements" ON client_measurements;

    CREATE POLICY "clients_select_own_measurements"
    ON client_measurements
    FOR SELECT
    TO authenticated
    USING (
      client_id IN (
        SELECT id FROM clients WHERE profile_id = auth.uid()
      )
    );
    ```

    **Paso 4 — Si el Paso 2 devolvió 0 filas, insertar datos de prueba:**
    ```sql

      ('24646591-53ec-4d1a-b92a-08f00e8d365b', NOW() - INTERVAL '20 days', 81.0, 17.8, 84, 94, 101, 37, 57, NULL),
    ```

    Si ya había datos (Paso 2 devolvió filas), saltar este paso.
