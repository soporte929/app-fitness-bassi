# Plan 30.1: Remove Direct Routine-to-Client Assignment Flows

## Objective
Eliminate all UI flows that allow a trainer to directly assign a Routine (workout_plan template) to a Client, bypassing the mandatory Routine тЖТ Plan тЖТ Client flow. After this plan, the only way to assign training to a client is through the `plans` module (`AssignPlanButton` тЖТ `assignPlanToClientAction` тЖТ `client_plans` table).

## Context
- `app/(trainer)/clients/[id]/page.tsx` тАФ Client detail page (imports both AssignRoutineButton and AssignPlanButton)
- `app/(trainer)/clients/[id]/assign-routine-button.tsx` тАФ Direct routine тЖТ client assignment button (**TO REMOVE**)
- `app/(trainer)/clients/[id]/assign-plan-button.tsx` тАФ Correct plan тЖТ client assignment (**KEEP**)
- `components/trainer/template-card.tsx` тАФ Routine template card with "Asignar a cliente" button
- `components/trainer/assign-template-modal.tsx` тАФ Modal triggered from template card to clone routine to client
- `components/trainer/assign-template-button.tsx` тАФ Standalone button wrapping AssignTemplateModal
- `app/(trainer)/routines-templates/actions.ts` тАФ Contains `clonePlanToClientAction` server action
- `app/(trainer)/routines-templates/page.tsx` тАФ Page listing templates (renders TemplateCard with clients)
- `app/(trainer)/plans/actions.ts` тАФ Contains `assignPlanToClient` (correct flow via client_plans table)
- `app/(trainer)/clients/actions.ts` тАФ Contains `assignPlanToClientAction` (correct flow via client_plans table)
