**Props:** `{ clientId: string }` — no dateStr needed (createNutritionFreeLogAction uses today's date internally).

**State:**
```typescript
type Step = 'input' | 'loading' | 'confirm' | 'fallback'
const [isOpen, setIsOpen] = useState(false)
const [step, setStep] = useState<Step>('input')
const [description, setDescription] = useState('')
const [estimate, setEstimate] = useState<MacroEstimate | null>(null)
const [manualKcal, setManualKcal] = useState('')
const [manualProtein, setManualProtein] = useState('')
const [manualCarbs, setManualCarbs] = useState('')
const [manualFat, setManualFat] = useState('')
const [errorMsg, setErrorMsg] = useState<string | null>(null)
const [isPending, startTransition] = useTransition()
```

**handleAnalyze():**
1. If `!description.trim()` show inline error and return.
2. `setStep('loading')` FIRST (before startTransition — ensures spinner appears immediately).
3. `startTransition(async () => { ... })` — call `parseNutritionAction(description)`.
4. On success: populate manual fields from estimate data, `setEstimate(result.data)`, `setStep('confirm')`.
5. On error: `setErrorMsg(result.error)`, `setStep('fallback')` — manual fields remain empty for user to fill.

**handleSave():**
```typescript
startTransition(async () => {
  await createNutritionFreeLogAction({
    clientId,
    foodName: estimate?.description || description,
    grams: null,
    calories: Number(manualKcal) || null,
    proteinG: Number(manualProtein) || null,
    carbsG: Number(manualCarbs) || null,
    fatG: Number(manualFat) || null,
  })
  closeModal()
})
```

**closeModal():** `setIsOpen(false)` then `setTimeout(300ms)` to reset all state after animation.

**Renders a trigger button** (NOT a FAB — this is an inline button to avoid conflicting with FoodSearchModal's FAB). The trigger is exported but NOT rendered inside this component's JSX return — it is a separate export OR the component just exports the modal and the page wires the trigger inline. Use the cleaner approach: the component renders ONLY the modal (overlay + bottom sheet). The trigger button is added directly in `page.tsx` as an inline button in the "Registro libre" section header.

So `AIFoodParserModal` receives `isOpen` and `onClose` props from the parent, OR uses internal state and exposes an open button:

**Preferred approach (simpler):** `AIFoodParserModal` manages its own `isOpen` state and renders both:
1. An inline button that sets `isOpen(true)` — this button is NOT a FAB; it's a small text/icon button meant to be placed in the Registro libre section header by the parent. Actually, the cleanest way: export a compound component where the trigger is inside the modal JSX, but positioned with `relative` in the section, NOT `fixed`.

**Final decision:** `AIFoodParserModal` renders:
- A small inline trigger button (not fixed, not a FAB) — e.g. `<button onClick={() => setIsOpen(true)} className="flex items-center gap-1 text-xs font-medium text-[#6b7fa3] hover:text-[#fb8500] transition-colors"><Sparkles className="w-3.5 h-3.5" />Analizar con IA</button>`
